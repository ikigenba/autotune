# Decision 4 — Scorer contract: external executable, hard failure

**Decision.** Package `internal/scorer` invokes the folder's user-supplied
`score` executable once per case:

- argv: `<folder>/score <case-dir>` (case dir absolute), working directory =
  the tune folder root, environment inherited untouched.
- stdin: the model's output text for that case.
- stdout: one JSON object `{"score": <float>, "feedback": "<optional>"}`.
  `score` is required and must satisfy `0 <= score <= 1`; `feedback` is
  optional free text handed to the improver's evidence bundle.

```go
package scorer

type Result struct {
    Score    float64
    Feedback string
}
type Scorer interface {
    Score(ctx context.Context, caseDir, modelOutput string) (Result, error)
}
func New(scorePath, workdir string) Scorer // exec-based implementation
```

Determinism is the scorer author's contract (stated in the `--init` stub and
docs); autotune enforces only the envelope. **Every scorer failure is a hard
abort of the run (exit 1)**: non-zero exit (stderr surfaced in the error),
stdout that is not valid JSON, a missing `score` field, or a score outside
[0,1]. Never score a broken case as 0 — a scorer that cannot be trusted
invalidates every number in the run.

This is a real external contract (process execution, pipes); its tests run
real script fixtures, not mocks.

**Rejected.** LLM-judge scoring (prior art deleted it as the core defect).
Built-in scorer types selected by config (couples autotune to tasks; a
reusable scorer-app library is a separate later project). Treating scorer
failure as score 0 (silent corruption of the accept arithmetic).

**Verification.** (all against real executable script fixtures)

- R-R06Q-Q6EE — the scorer executes `score` with the case dir as its single
  argument, cwd = the tune folder, and the model output delivered on stdin
  (fixture echoes back what it received to prove all three).
- R-R1EN-3Y53 — a valid stdout JSON yields the parsed score and optional
  feedback; absent `feedback` yields an empty string.
- R-R2MJ-HPVS — unparseable stdout, a missing `score` field, or a score
  outside [0,1] (e.g. 1.5, -0.1) each return an error; the run maps scorer
  errors to a hard abort with exit 1.
- R-R3UF-VHMH — a non-zero scorer exit returns an error carrying the
  process's stderr text.
