# Decision 2 — Config model: two agents, namespaced `-c`, folder pins

**Decision.** Package `internal/config` resolves two independent agent
configurations — **runner** (executes the prompt under test) and **improver**
(proposes revisions) — from three layers, later wins per key:

1. built-in defaults (per section),
2. the folder's `config.json` (`runner`/`improver` objects of string
   key→value pairs),
3. repeatable CLI `-c <section>.<key>=<value>` overrides in argv order.

```go
package config

type Section struct {
    Provider, Model, Auth, AuthFile string
    Temperature, TopP               *float64
    MaxTokens                       int
    Reasoning                       agentkit.ReasoningValue
    Retry                           agentkit.RetryPolicy
    BaseURL                         string
}
type Config struct{ Runner, Improver Section }

func ParsePair(raw string) (key, value string, err error) // split on first "="
func Resolve(configRaw []byte, cliPairs []string, getenv func(string) string, home string) (*Config, error)
func (s *Section) Conversation(system string) (*agentkit.Conversation, error) // builds provider + conv
```

Key vocabulary (agent-repl's, minus keys autotune owns itself): `provider`,
`model`, `auth`, `auth_file`, `temperature`, `top_p`, `max_tokens`, `effort`,
`thinking_budget`, `thinking`, `base_url`, `max_attempts`, `base_delay`,
`max_delay`, `max_elapsed`, `ignore_retry_after`. `system` is **not** a key:
the runner's system prompt is always `prompt.txt`, the improver's always
`improve.md`.

Rules, all boundary-validated with loud errors:

- A `-c` value without `=`, or with an empty key or value, is a usage error.
- A CLI key **must** carry a `runner.` or `improver.` prefix; an unprefixed
  key is a usage error stating the required namespaces (explicit over
  implicit; no "applies to both"). Keys inside `config.json` are unprefixed —
  the section object supplies the namespace.
- An unknown key (either layer) is an error naming the key.
- `auth` accepts only `key` | `sub`. `key`: the provider's API key is read
  from its conventional env var via the injected `getenv` at conversation
  build time. `sub`: OAuth token loaded from `auth_file`, defaulting to
  `<home>/.autotune/auth.json` (the product's shared login file), via
  agentkit's subscription store; the file is produced by the external
  `oauth-login` tool.
- `temperature`/`top_p` parse as floats, `max_tokens`/`thinking_budget`/
  `max_attempts` as ints, `base_delay`/`max_delay`/`max_elapsed` as Go
  durations; `effort` maps to `agentkit.Level`, `thinking_budget` to
  `agentkit.Budget`, `thinking=false` to `agentkit.DisableReasoning()`.
  Built-in defaults are the zero config: provider/model per the scaffold
  template only enters via `config.json`; `Resolve` itself requires
  `provider` and `model` to be set for each section after merging and errors
  otherwise.

**Rejected.** Unprefixed `-c` keys applying to both sections (invites
accidentally un-pinning the runner). Dedicated flags per parameter
(`--runner-model`) — breaks the household `-c` convention. Supporting
`system` as a key (the folder owns both system prompts by definition).

**Verification.**

- R-QMRU-IP8R — layered precedence: a key set in defaults, overridden in
  `config.json`, then overridden again by `-c`, resolves to the `-c` value;
  with two `-c` pairs for the same key the later one wins.
- R-QNZQ-WGZG — `ParsePair` splits on the first `=` (value may contain `=`);
  missing `=`, empty key, or empty value each yield an error, surfaced as a
  usage error (exit 2).
- R-QQFJ-O0GU — an unprefixed CLI key (e.g. `-c model=x`) is rejected with
  an error naming the `runner.`/`improver.` namespaces; `runner.` pairs
  never affect the improver section and vice versa.
- R-QRNG-1S7J — an unknown key in either `-c` or `config.json` is rejected
  with an error naming the offending key.
- R-QSVC-FJY8 — vocabulary mapping: representative keys land in the right
  `Section`/agentkit fields with correct types — `temperature=0` →
  `*float64(0)` in `Gen`, `max_tokens` → int, `base_delay=2s` → duration in
  `Retry`, `effort=high` → `agentkit.Level`, `thinking=false` → disabled
  reasoning — and a malformed number/duration is a boundary error.
- R-QU38-TBOX — `auth` accepts exactly `key`|`sub` (anything else errors);
  with `sub` and no explicit `auth_file` the resolved path is
  `<home>/.autotune/auth.json`; after merging, a section missing `provider`
  or `model` is an error naming the section.
