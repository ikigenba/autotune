# Decision 6 — Improver: fresh-context bare call, evidence bundle

**Decision.** Package `internal/improver` produces one candidate prompt per
iteration with a single bare agentkit call — fresh conversation every time,
strictly sequential, no tools. System prompt = the folder's `improve.md`.
User message = an evidence bundle autotune assembles:

1. the incumbent (current best) prompt, fenced;
2. score summary: baseline, current best, epsilon, and a per-case score
   table (name, score);
3. the worst-K cases in full — input, model output, score, scorer feedback
   (K = `--worst` default 5; ties broken by case name);
4. attempt history: one line per prior candidate — its change summary, its
   composite, accept/reject — so failed theories are not re-proposed
   (attempt memory without chat memory).

Response contract: the reply must carry a change-summary line and exactly one
fenced block whose contents are the complete replacement prompt. The candidate
is the fence contents verbatim; the summary feeds `history.md` and future
bundles.

**Lenient parse.** The `SUMMARY:` marker is matched tolerantly — leading
whitespace and surrounding markdown emphasis (`**`, `*`, `_`, `__`, backticks)
are stripped, so `**SUMMARY:** …` and an indented marker both parse — while the
contract stays unambiguous: exactly one summary line **outside** the fence, and
exactly one fenced block. A `SUMMARY:`-looking line **inside** the fence is part
of the prompt, never the response summary. Only genuinely ambiguous shapes
(missing summary, missing fence, more than one fence) still fail the parse.

```go
package improver

type Attempt struct{ Summary string; Composite float64; Accepted bool }
type Evidence struct {
    Incumbent string
    Baseline, Best, Epsilon float64
    Cases    []runner.CaseResult // full set; bundle renders table + worst-K
    History  []Attempt
    WorstK   int
}
var ErrUnparseable = errors.New("improver response unparseable")
func BuildBundle(e Evidence) string
func Parse(reply string) (summary, prompt string, err error)
func Propose(ctx context.Context, nc runner.NewConv, improveMD string, e Evidence, maxRetries int, warn runner.WarnFunc) (summary, prompt string, err error)
```

Each drained stream's `Warnings()` are passed to `warn` (nil = dropped),
same reporting contract as the runner (D03); the surfacing policy is D05's.

**Corrective retry.** A parse failure is retried up to `--max-retries`
(default 3), each with a **fresh** conversation (no chat memory, preserving the
fresh-context design), but the retry's user message is **prefixed with the
prior parse error** and a restatement of the required format. A model that
would deterministically mis-format a given bundle is told what was wrong
instead of being handed the identical bundle, so the retry loop can converge.
If every attempt still fails, `Propose` returns an error wrapping the sentinel
`ErrUnparseable` (its message names the attempt count and the last parse
error). `ErrUnparseable` marks a **recoverable** exhaustion — the loop (D05)
skips the iteration and continues; it is distinct from a hard call or context
error, which still stops the run. Holdout cases never appear in `Evidence` by
construction — the improver physically cannot see what it is not handed.

**Rejected.** Agentic improver with file tools over a workspace (more power,
but drags in a protocol and a holdout-leak surface). Accumulating one long
improver context across iterations (prior art: fresh context per attempt,
with explicit attempt history, converges better than a chat of failed
theories). Passing all cases in full (context bloat for no gain — hence
worst-K).

**Verification.**

- R-RETJ-BFAQ — the rendered bundle contains the incumbent prompt, the
  baseline/best/epsilon numbers, a score row for every dev case, full
  detail (input, output, score, feedback) for exactly the K worst cases,
  and one history line per prior attempt with its summary/score/verdict;
  holdout content never appears.
- R-RG1F-P71F — `Parse` extracts the summary line and the fenced prompt
  verbatim (including trailing/leading whitespace inside the fence) and is
  lenient about the marker: leading whitespace and surrounding markdown
  emphasis (`**SUMMARY:**`, `*…*`, `__…__`) parse, and a `SUMMARY:`-looking
  line inside the fence is not counted as the summary; replies missing
  `SUMMARY:`, with an empty summary, missing a fence, with two fences, or with
  two out-of-fence summary lines each error.
- R-RH9C-2YS4 — a malformed reply triggers a fresh retry call whose user
  message carries the prior parse error forward (corrective, not a blind
  identical resend), while remaining a fresh conversation with no carried chat
  history; exhausting `maxRetries` returns an error that wraps `ErrUnparseable`
  (so the loop can recognize it as recoverable) and names the attempt count.
- R-RIH8-GQIT — proposals are sequential and fresh: each `Propose` builds a
  new conversation via the seam and never reuses prior history.
