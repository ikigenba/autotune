# Decision 10 — Versioning, release, and install tooling (embed's scheme)

**Decision.** autotune adopts the release/versioning scheme of the sibling
`embed` project wholesale (see research.md, "embed — release/versioning
scheme", for the verified reference). Product owns the contractual constants
(semver tags with leading `v`, starting `v0.1.0`; unreleased builds are
`dev`); this Decision owns the mechanism.

**Version stamping.** `cmd/autotune/main.go` declares
`var version = "dev"` and passes it into `app.Run` as `Deps.Version` (D09).
The variable is stamped at link time via `-X main.version=…`:

- Local Makefile build: `$(git describe --tags --always --dirty)`.
- goreleaser release build: `{{ .Tag }}`.

Both carry the leading `v`, so `-V` output is identical whether the binary
came from `make` or a release. The hardcoded `Version` constant in
`internal/app` is deleted; `main` is the only version authority.

**Makefile** — embed's, with `autotune` substituted:

- Vars: `BINARY := autotune`, `BIN_DIR := bin`, `PREFIX ?= $(HOME)/.local`.
- `build`: file-level rule producing `bin/autotune` (deps: `go.mod`,
  `go.sum`, `$(shell find cmd internal -name '*.go')`), stamping
  `-ldflags "-X main.version=$(shell git describe --tags --always --dirty)"`.
- `fmt`: `go fmt ./...`. `test`: `go test ./...`.
- `install`: `build` then `install -d $(PREFIX)/bin` +
  `install -m 0755 bin/autotune $(PREFIX)/bin/autotune`.
- `clean`: `rm -rf bin` + `go clean`.
- `.gitignore` gains `bin/`.

**Release = tag push.** Cutting a release is
`git tag vX.Y.Z && git push --tags`, done manually by the operator — never
by any build loop. Two files, embed's verbatim with `embed` → `autotune`
and repo `ikigenba/autotune` substituted:

- `.github/workflows/release.yml`: on push of `v*` tags; checkout with
  `fetch-depth: 0`; `actions/setup-go@v5` (Go 1.26);
  `goreleaser/goreleaser-action@v6`, `args: release --clean`;
  `permissions: contents: write`.
- `.goreleaser.yaml` (v2): one build from `./cmd/autotune`,
  `CGO_ENABLED=0`, linux+darwin × amd64+arm64, ldflags
  `-s -w -X main.version={{ .Tag }}`; tar.gz archives with versionless
  asset names (`name_template: "{{ .Binary }}_{{ .Os }}_{{ .Arch }}"`) so
  `releases/latest/download/autotune_<os>_<arch>.tar.gz` is a stable URL;
  `checksums.txt`; GitHub-generated changelog.

**install.sh** — embed's verbatim at the repo root, substitutions:
`REPO="ikigenba/autotune"`, `BINARY="autotune"`, pin var
`AUTOTUNE_VERSION`. Behavior: `set -eu`; OS/arch detection
(linux/darwin, amd64/arm64, loud failure otherwise); fetch latest or the
pinned tag; install to `${BINDIR:-${PREFIX:-$HOME/.local}/bin}`; verify by
running `-V`; warn when the destination is off `PATH`. Invoked as
`curl -fsSL https://raw.githubusercontent.com/ikigenba/autotune/main/install.sh | sh`.

**README `## Install`** — embed's section shape: the curl one-liner, the
`BINDIR`/`PREFIX`/`AUTOTUNE_VERSION` override notes, and the
`make install` from-source path.

The workflow, goreleaser config, install.sh, and README section are
structural artifacts: their proof is deterministic file checks in the
realizing phase's Done-when, not requirement ids — the GitHub Actions and
goreleaser runtime contracts cannot be exercised by the local suite, and
exercising a real tag push from a test would publish a release. The
stamping behavior, which the local suite *can* prove against the real
compiled binary, carries the ids below.

**Rejected.** Keeping the hardcoded `const Version` in `internal/app`
(cannot be stamped; drifts from tags). Name-prefixed version output
(`autotune dev`) — embed prints the bare version; identical semantics was
the stated goal. A Makefile-driven release (goreleaser+Actions is the
household scheme; local `make` never publishes). Versioned release asset
names (breaks the stable `latest/download` URL install.sh depends on).

**Verification.**

- R-9A0E-BG7N — a binary built by the test with plain `go build` (no
  ldflags) prints exactly `dev` on stdout and exits 0 for both `-V` and
  `--version`.
- R-9B8A-P7YC — a binary built by the test with
  `-ldflags "-X main.version=<sentinel>"` prints exactly the sentinel on
  stdout and exits 0 for `-V`; the substrate is the real compiled binary,
  proving the linker-stamp path release builds rely on.
