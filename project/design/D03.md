# Decision 3 — Runner: one bare call per case, bounded parallelism

**Decision.** Package `internal/runner` evaluates a candidate prompt over a
case set. Each case is one bare agentkit call — no tools, fresh
`agentkit.Conversation` per case (agentkit conversations are single-use here
and not concurrency-safe): `System` = the candidate prompt text, user message
= the case's `input.txt` contents, output = the final assistant text
(`MessageDone`), usage/cost taken from the drained stream.

```go
package runner

type CaseResult struct {
    Case   folder.Case
    Output string
    Score  float64  // filled by the scorer (D04)
    Feedback string
    Cost   agentkit.Cost
}
type EvalResult struct {
    Cases     []CaseResult // sorted by case name
    Composite float64      // mean of case scores
    Cost      agentkit.Cost
}
// NewConv is the seam: builds a fresh conversation for one call
// (config.Section.Conversation under the hood; a scripted fake in tests).
type NewConv func(system string) (*agentkit.Conversation, error)

// WarnFunc receives every provider warning from a drained stream; nil drops.
type WarnFunc func(agentkit.Warning)

func Evaluate(ctx context.Context, nc NewConv, score scorer.Scorer,
    prompt string, cases []folder.Case, parallel int, warn WarnFunc) (*EvalResult, error)
```

- Cases run concurrently on a worker pool bounded by `parallel` (CLI
  `--parallel`, default 4). Results are collected and **sorted by case name
  before aggregation**, so the scorecard is byte-identical regardless of
  completion order. The scorer is invoked per case as its call completes.
- The composite is the arithmetic mean of case scores.
- Any terminal model-call failure (stream error after agentkit's own
  retries) or scorer failure cancels the remaining work and fails the whole
  evaluation — no partial scorecards, ever (a 14-of-15 composite is not
  comparable to a 15-of-15 incumbent). The run maps this to exit 1.
- Costs are summed into `EvalResult.Cost` and fed to the loop's spend rail.
- After each stream is drained, its `Warnings()` are passed one by one to
  `warn` (nil = dropped). The surfacing policy — where they land, dedup —
  is D05's; the runner only reports.

**Rejected.** Sequential evaluation (wall-clock dominant cost; calls are
independent). Scoring a failed case as 0 and continuing (corrupts the accept
arithmetic silently). Reusing one conversation across cases (history bleed;
agentkit forbids concurrent use anyway).

**Verification.**

- R-QVB5-73FM — each case's call uses a fresh conversation with `System` set
  to the candidate prompt and the user message equal to that case's
  `input.txt`; the captured output is the final assistant text.
- R-QWJ1-KV6B — with `parallel` &lt; case count, no more than `parallel`
  calls are in flight at once, and the returned `Cases` slice and composite
  are identical regardless of completion order (deterministic aggregation).
- R-QXQX-YMX0 — a terminal failure in any single case's call fails the whole
  `Evaluate` with an error identifying the case, and in-flight/pending work
  is cancelled; the loop maps this to exit 1.
- R-QYYU-CENP — per-case costs are summed into `EvalResult.Cost` (fake
  provider reports known costs; the sum is exact).
