# Decision 8 — CLI surface, exit codes, health-log output

**Decision.** Package `internal/cli` owns argument parsing and the rendered
health log.

Usage:

```
autotune --init <folder>
autotune [flags] <folder>

  -c section.key=value   repeatable config override (D02)
  --repeat N             baseline repeats -> epsilon        (default 3)
  --parallel N           concurrent case evaluations        (default 4)
  --worst N              worst-K cases in improver bundle   (default 5)
  --max-retries N        malformed-improver-reply retries   (default 3)
  --max-iterations N     0 = unlimited                      (default 0)
  --max-time D           e.g. 30m, 2h; 0 = unlimited
  --max-spend USD        0 = unlimited
  -h | --help            usage; -V | --version              version
```

Parsing is hand-rolled (ralph's pattern): both `--flag value` and
`--flag=value` accepted; exactly one positional folder argument required
(except `--help`/`--version`); unknown flag, missing folder, or malformed
value → usage message on stderr, exit 2.

```go
package cli

type Options struct {
    Init bool; Folder string; Config []string
    Repeat, Parallel, Worst, MaxRetries, MaxIterations int
    MaxTime time.Duration; MaxSpend float64
    Help, Version bool
}
func Parse(args []string) (Options, error)

type StopReason int // maps 1:1 to exit codes
const (Done StopReason = iota; Failed; Usage; RailCrossed; Interrupted)
func (r StopReason) ExitCode() int // 0,1,2,3,130
```

**Health log** (stdout, line-oriented — "a log, to know it's healthy"):
every meaningful event emits one line with numbers: per-baseline-repeat
composites, the baseline/epsilon line with case and call counts and spend;
per iteration an improving line and an `evaluating i/N` progress counter,
then the composite with `ACCEPT`/`reject` and the threshold it had to beat
plus cumulative elapsed/spend (and rail ceilings when set); finalize lines
(holdout, verdict); the winner path and a unified diff `prompt.txt` →
winner. On a TTY the `evaluating i/N` counter ticks in place; when piped,
plain appended lines; color only on a TTY and never when `NO_COLOR` is set.
Everything logged is also captured in `history.md`/`summary.md` (D07), so
stdout is ephemeral. TTY detection and streams come through `Deps`.

**Rejected.** stdlib `flag` (no `--flag value` doubles support for the
ralph-style long flags; hand-rolled matches the household). Subcommands
(`autotune init`) — `--init` as a mode flag matches the sibling tools' flat
surface. `--format jsonl` in v1 (nothing consumes it yet; add when
something does).

**Verification.**

- R-RR0J-54PO — `Parse` accepts both `--flag value` and `--flag=value`
  forms for every flag, repeatable `-c`, and the documented defaults
  (repeat 3, parallel 4, worst 5, max-retries 3, rails 0); `--max-time`
  takes Go duration syntax.
- R-RS8F-IWGD — unknown flag, missing/extra positional, or malformed flag
  value each produce a usage error on stderr and exit 2; `--help` and
  `--version` print to stdout and exit 0.
- R-RTGB-WO72 — `StopReason.ExitCode` maps Done→0, Failed→1, Usage→2,
  RailCrossed→3, Interrupted→130.
- R-RUO8-AFXR — a full run's stdout (piped, non-TTY) contains the baseline
  repeats, the baseline/epsilon line, one line per iteration with composite
  and ACCEPT/reject verdict and cumulative spend, the finalize verdict, and
  a unified diff when a winner exists.
- R-RVW4-O7OG — with a non-TTY stdout or `NO_COLOR` set, output contains no
  ANSI escape sequences; the in-place ticker is emitted only on a TTY.
