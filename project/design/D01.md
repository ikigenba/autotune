# Decision 1 — Tune-folder contract and `--init`

**Decision.** A tune folder is the unit autotune operates on. Package
`internal/folder` owns loading/validating it and scaffolding it.

Required layout (paths relative to the folder):

```
prompt.txt          # the prompt under test (incumbent starting prompt)
improve.md          # system prompt for the improver
score               # user-supplied executable, any language, +x required
config.json         # pinned agent config: {"runner": {...}, "improver": {...}}
cases/dev/<case>/input.txt      # >=1 dev case; other files are the scorer's
cases/holdout/<case>/input.txt  # optional; may be empty (skips holdout check)
runs/               # created on demand by runs; gitignored
.gitignore          # ships with the scaffold
```

```go
package folder

type Case struct{ Name, Dir, Input string }
type Folder struct {
    Root       string
    Prompt     string   // contents of prompt.txt
    ImproveMD  string   // contents of improve.md
    ScorePath  string   // absolute path to score
    ConfigRaw  []byte   // raw config.json for the config package
    Dev        []Case   // sorted by Name
    Holdout    []Case   // sorted by Name; may be empty
}
func Load(root string) (*Folder, error) // validates everything above
func Init(root string) error            // scaffolds; refuses non-empty
```

`Load` fails with an error naming the first missing/invalid piece: absent
`prompt.txt`, `improve.md`, `config.json`, `score`, a `score` without the
executable bit, or zero dev cases. A case is any directory under `cases/dev`
(or `cases/holdout`) containing `input.txt`; cases are sorted by name.

`Init` writes the full skeleton into an empty or freshly-created directory:
`prompt.txt`, `improve.md`, an executable `score` stub (`#!/usr/bin/env bash`
emitting a placeholder score with an explanatory comment), `config.json`
template, `cases/dev/example/input.txt`, `cases/holdout/.gitkeep`, and
`.gitignore` containing `runs/`. Template `config.json` (using the product's
contractual model constants):

```json
{
  "runner":   {"provider": "openai", "model": "gpt-5.6-luna", "auth": "sub", "temperature": "0"},
  "improver": {"provider": "openai", "model": "gpt-5.6-sol",  "auth": "sub", "effort": "high"}
}
```

All template values are strings, in the same `key=value` vocabulary as `-c`
(D02); `config.json` is exactly "a bag of pre-applied `-c` pairs per
section".

**Rejected.** A manifest file naming the cases (the filesystem is the
manifest; less to keep in sync). Free-form prompt filename via config
(convention over configuration; the fixed name is the contract). Scaffolding
into a non-empty directory with per-file skip logic (refusal is simpler and
safer).

**Verification.**

- R-QFGG-82SL — `Init` on an empty (or nonexistent, creatable) directory
  creates exactly the skeleton set: `prompt.txt`, `improve.md`, executable
  `score`, `config.json`, `cases/dev/example/input.txt`,
  `cases/holdout/.gitkeep`, `.gitignore`; no other entries.
- R-QGOC-LUJA — `Init` on a directory containing any entry fails without
  writing anything, and the CLI maps this to a usage error (exit 2).
- R-QHW8-ZM9Z — the scaffolded `config.json` parses under D02's loader and
  yields runner `provider=openai, model=gpt-5.6-luna, auth=sub,
  temperature=0` and improver `provider=openai, model=gpt-5.6-sol, auth=sub,
  effort=high`.
- R-QJ45-DE0O — the scaffolded `.gitignore` covers `runs/`, and the
  scaffolded `score` has the executable bit set.
- R-QKC1-R5RD — `Load` on a folder missing any required piece (each of:
  `prompt.txt`, `improve.md`, `config.json`, `score`, executable bit on
  `score`, at least one dev case) fails with an error naming that piece;
  an empty `cases/holdout` is not an error.
- R-QLJY-4XI2 — `Load` returns dev and holdout cases sorted by case name
  with `input.txt` contents populated, and ignores non-directory entries
  under `cases/`.
