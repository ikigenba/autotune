# Decision 9 — Composition root, seams, end-to-end proof

**Decision.** `cmd/autotune/main.go` is a thin composition root, agent-repl's
pattern: `main()` builds the real `Deps` and `os.Exit(run(...))`.

```go
// internal/app
type Deps struct {
    Stdout, Stderr io.Writer
    Getenv         func(string) string
    Now            func() time.Time
    Home           string
    IsTTY          bool
    Version        string                 // from cmd/autotune's var version (D10)
    NewProvider    config.ProviderFactory // real: agentkit provider subpackages
}
func Run(ctx context.Context, deps Deps, args []string) int
```

`cmd/autotune/main.go` declares `var version = "dev"` (the linker-stamp
target, D10) and passes it as `Deps.Version`; `app` holds no version
constant of its own.

`Run` wires the packages: `cli.Parse` → (`folder.Init` | `folder.Load` →
`config.Resolve` → `workspace.Create` → `loop.Run`). Signal handling via
`signal.NotifyContext(SIGINT, SIGTERM)`. All logic is reachable in-process
with fake deps; the only code outside the seam is `main()` itself.

Test substrates, stated explicitly:

- The provider HTTP contract is agentkit's to prove (its own suite);
  autotune's boundary is the `agentkit.Provider` interface, driven by a
  scripted fake in tests.
- The scorer subprocess contract is autotune's and is proven against real
  executable fixtures (D04).
- The whole pipeline is proven by an end-to-end test of `app.Run` with a
  fake provider, a real temp tune folder, and a real scorer script: init,
  full loop with accepts and rejects, finalize — real filesystem, real
  subprocesses, fake network.
- The compiled binary's CLI envelope (`--init`, `--help`, `--version`) is
  proven by building and executing the real binary.

**Rejected.** A hidden "scripted provider" mode inside the shipped binary
for binary-level loop tests (test machinery in the product; the in-process
seam plus a binary envelope check covers it without shipping fakes).

**Verification.**

- R-RX41-1ZF5 — end-to-end through `app.Run` (in-process, scripted fake
  provider, real temp folder scaffolded by `--init` then filled, real
  scorer script): a run that accepts at least one candidate and rejects at
  least one exits 0, and the workspace holds the config stamp, baseline,
  all candidates, promoted best, history, and a summary with a holdout
  verdict.
- R-RYBX-FR5U — the compiled `autotune` binary (built by the test via
  `go build`): `--help` exits 0 with usage on stdout; `-V`/`--version`
  exit 0 with the bare version string on stdout; `--init` into a temp dir
  creates the D01 skeleton; a usage error (unknown flag) exits 2 with the
  message on stderr. (What the version string contains, stamped and
  unstamped, is D10's ids.)
