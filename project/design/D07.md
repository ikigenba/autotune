# Decision 7 — Run workspace: `runs/<id>/`, incumbent never touched

**Decision.** Package `internal/workspace` owns the durable record of one
invocation, under `<folder>/runs/<run-id>/` where `run-id` is the UTC start
time formatted `20060102-150405` (from the injected clock).

```
runs/20260722-141502/
  config.json          # the fully resolved config stamp (both sections)
  baseline.json        # composites per repeat, baseline, epsilon
  candidates/001-prompt.txt        # every candidate, accepted or not
  candidates/001-scorecard.json    # its per-case scores + composite
  best/prompt.txt      # current incumbent (starts as copy of prompt.txt)
  best/scorecard.json
  history.md           # one line per attempt: NNN, summary, score, verdict
  summary.md           # written at finalize
```

```go
package workspace

type Workspace struct{ Dir string }
func Create(folderRoot string, now time.Time) (*Workspace, error)
func (w *Workspace) WriteConfigStamp(cfg *config.Config) error
func (w *Workspace) WriteBaseline(composites []float64, baseline, epsilon float64) error
func (w *Workspace) WriteCandidate(n int, prompt string, ev *runner.EvalResult) error
func (w *Workspace) PromoteBest(n int) error             // copies candidate over best/
func (w *Workspace) AppendHistory(line string) error

// Summary is workspace-owned so the package never imports the loop
// (loop depends on workspace, not the reverse).
type Summary struct {
    Baseline, Epsilon, Best float64
    Accepted                int
    Holdout                 *float64
    Verdict, StopReason     string
}
func (w *Workspace) WriteSummary(s Summary, diff string) error
```

Invariants:

- **`<folder>/prompt.txt` is never written by any code path.** Adoption is
  the operator copying the diff in; the tool only reports.
- `best/` changes only on accept (candidate files copied over it); rejects
  leave it untouched.
- Every candidate is recorded whether accepted or not; `history.md` is
  append-only within a run; scorecard JSON is written with sorted cases and
  fixed float formatting so identical results are byte-identical.
- `runs/` is per-invocation; no resume — a new invocation always creates a
  new run dir.

**Rejected.** Workspace under `~/.autotune/` (a tune folder should be
self-describing: prompt, cases, scorer, and tuning history travel together).
Auto-writing the winner to `prompt.txt` (prior art's manual-adoption rule is
what made accepted results trustworthy). Overwriting a single `latest/` dir
(history is the point).

**Verification.**

- R-RJP4-UI9I — `Create` makes `runs/<UTC-timestamp>/` (from the injected
  clock) with `candidates/` and `best/` present, and the config stamp and
  baseline files are written with the resolved values.
- R-RM4X-M1QW — every attempt persists `candidates/NNN-prompt.txt` and
  `NNN-scorecard.json` (zero-padded, sequential), for rejects as well as
  accepts; scorecard output is byte-deterministic (sorted cases, fixed
  float formatting).
- R-RNCT-ZTHL — accept copies the candidate's prompt and scorecard over
  `best/`; a reject leaves `best/` byte-identical.
- R-ROKQ-DL8A — `history.md` gains exactly one line per attempt carrying
  the attempt number, change summary, composite, and accept/reject verdict.
- R-RPSM-RCYZ — after a full run (accepts and rejects, finalize included)
  `<folder>/prompt.txt` is byte-identical to its pre-run contents, and
  `summary.md` records baseline, best, epsilon, holdout result/verdict (or
  their absence).
